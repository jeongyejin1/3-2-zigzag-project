## 1. 문제 인식

패션 커머스 플랫폼 지그재그에는 하루에도 수만 건의 의류 리뷰가 쏟아진다. 소비자들은 "이 옷의 핏이 나한테 맞을까?", "재질이 까슬거리지는 않을까?"를 고민하며 수많은 리뷰를 탐색하지만, 개인이 수천 개의 리뷰를 일일이 읽고 쇼핑몰별 특징을 파악하기는 불가능에 가깝다. 이는 결국 구매 결정의 피로도로 이어진다.

판매자(쇼핑몰) 입장에서도 마찬가지다. 매출 데이터만으로는 알 수 없는 **'고객이 구체적으로 무엇에 만족하고, 무엇에 불만족하는지'**에 대한 정성적인 피드백을 한눈에 파악하기 어렵다. 예를 들어, "바지 맛집"으로 소문난 쇼핑몰이라도 구체적으로 '기장'이 좋은지 '스판'이 좋은지, 혹은 '보풀'이 문제인지 알기 위해서는 방대한 텍스트 데이터를 분석해야 한다.

본 프로젝트는 이러한 문제를 해결하기 위해 **자연어 처리(NLP) 기술**을 도입했다. 지그재그 상위 8개 쇼핑몰의 리뷰 데이터를 **Kiwi 형태소 분석기**와 **LDA 토픽 모델링**으로 분석하여, 각 쇼핑몰의 강점(긍정 토픽)과 약점(부정 토픽)을 워드클라우드로 시각화함으로써 소비자와 판매자 모두에게 직관적인 인사이트를 제공하고자 한다.

## 2. 프로젝트 목표

본 프로젝트의 핵심 목표는 다음과 같다.

- **첫째, 쇼핑몰별 리뷰 분석 파이프라인을 자동화한다.** 상품 URL과 ID를 기반으로 리뷰 데이터를 쇼핑몰별로 자동 분류하고, 전처리부터 시각화까지의 과정을 하나의 스크립트(`zigzag_full_analysis.py`)로 구현한다.
- **둘째, 빠르고 정확한 한국어 텍스트 분석 환경을 구축한다.** 기존 자바(JVM) 기반 분석기(Okt)의 속도 문제와 환경 오류를 개선하기 위해, 순수 Python 기반의 **Kiwi(Kiwi-Piepy)** 분석기를 도입하여 형태소 분석의 정확도와 속도를 높인다.
- **셋째, 쇼핑몰별 핵심 키워드(Topic)를 시각화한다.** 단순한 감성 분류(긍정/부정)를 넘어, **LDA(Latent Dirichlet Allocation)** 기법을 통해 해당 쇼핑몰 리뷰에서 가장 가중치가 높은 핵심 단어를 추출하고, 이를 워드클라우드로 그려내어 브랜드 이미지를 한눈에 파악하도록 한다.

## 2.1. 분석 대상 쇼핑몰

지그재그 플랫폼 내에서 리뷰 수가 많고 인지도가 높은 상위 8개 쇼핑몰을 분석 대상으로 선정했다.

| 쇼핑몰 | 스타일/특징 | 쇼핑몰 | 스타일/특징 |
|---|---|---|---|
| **슬로우앤드** | 감성 베이직, 니트/셔츠 강점 | **베니토** | 오피스룩, 페미닌 무드 |
| **고고싱** | 가성비 바지/슬랙스 특화 | **베이델리** | 데일리, 러블리 캐주얼 |
| **블랙업** | 유니크, 힙(Hip)한 스타일 | **어텀** | 심플, 모던 베이직 |
| **육육걸즈** | 다양한 사이즈, 편안한 핏 | **핫핑** | 빅사이즈 포함 폭넓은 옵션 |

## 3. 전체적인 프로젝트 과정

데이터 로드부터 형태소 분석, 토픽 모델링, 시각화 저장까지의 전 과정을 Python 스크립트로 통합하여 실행한다.

| 단계 | 주요 기능 | 설명 |
|:---:|---|---|
| **1** | **Data Loading & Mapping** | `url` 정보를 파싱하여 상품 ID를 추출하고, 사전 정의된 Mapping Dictionary를 이용해 8개 쇼핑몰로 데이터를 분류한다. |
| **2** | **Preprocessing (Kiwi)** | **(핵심 기술)** `Kiwi` 분석기를 사용하여 텍스트에서 불용어를 제거하고, 의미 있는 명사(Noun)만을 추출하여 분석 품질을 높인다. |
| **3** | **Topic Modeling (LDA)** | `CountVectorizer`로 단어를 벡터화한 후, LDA 알고리즘을 통해 긍정/부정 그룹별로 가장 강력한 주제(Topic) 단어를 도출한다. |
| **4** | **Visualization** | 추출된 키워드를 바탕으로 **긍정(Blue 계열)**과 **부정(Red 계열)** 워드클라우드 이미지를 생성하고 파일로 자동 저장한다. |

## 4. 데이터 수집 및 전처리

### 4.1. 리뷰 데이터 수집 및 불균형 문제 인식

지그재그 플랫폼 내 상위 8개 쇼핑몰에서 총 **22,901개**의 원시 리뷰 데이터를 수집하였습니다. 수집된 데이터를 분석한 결과, 5점 만점 리뷰가 압도적으로 많은 **클래스 불균형(Class Imbalance)** 현상이 확인되었습니다.

* **총 수집 데이터:** 22,901건
* **별점 분포:** 5점 리뷰가 전체의 약 84%(19,222건)를 차지하며, 1~2점대의 부정 리뷰는 극소수에 불과함.

이러한 불균형한 데이터를 그대로 학습시킬 경우, 모델이 무조건 '긍정'으로만 예측하는 편향(Bias) 문제가 발생할 수 있습니다. 따라서 정확한 감성 분석을 위해 **데이터 정제 및 균형 맞추기 작업**이 필수적이었습니다.

### 4.2. 데이터 정제 및 라벨링 (Labeling)

단순히 별점(Score)만을 기준으로 긍정/부정을 나누는 것은 정확하지 않습니다. (예: 별점은 5점이지만 "배송이 너무 늦었어요"라고 적는 경우). 따라서 다음과 같은 기준을 통해 데이터의 품질을 높였습니다.

1.  **노이즈 제거:** "좋아요", "굿" 등 너무 짧거나 의미 없는 단답형 리뷰 제거.
2.  **라벨링 기준 재정립:**
    * **부정(0):** 1~3점 리뷰 포함 및 내용 기반(Content-based) 필터링을 통해 불만 족 키워드(반품, 실밥, 냄새 등)가 포함된 리뷰 선별.
    * **긍정(1):** 4~5점 리뷰 중 구체적인 만족 사유(핏, 재질 등)가 포함된 양질의 데이터 선별.

### 4.3. 최종 데이터셋 구축 (Balancing)

모델의 학습 성능을 극대화하기 위해 **언더샘플링(Under-sampling)** 기법을 적용하여 긍정 데이터와 부정 데이터의 비율을 1:1로 맞췄습니다.

* **데이터 통합:** 각 쇼핑몰별로 정제된 데이터를 하나로 병합.
* **최종 구성:** 부정 리뷰(1,798개)와 긍정 리뷰(1,798개)를 동수로 맞춰 총 **3,596개**의 고품질 학습 데이터셋을 구축하였습니다.

## 6. 최종 결론 및 인사이트

본 프로젝트를 통해 지그재그 상위 8개 쇼핑몰의 리뷰 데이터를 정량적, 시각적으로 분석한 결과 다음과 같은 핵심 인사이트를 도출할 수 있었다.

### 6.1. 핵심 인사이트

1.  **카테고리별 '킬러 콘텐츠'의 중요성**:
    * **고고싱(바지/기장)**, **슬로우앤드(색감/니트)** 처럼 각 쇼핑몰이 가진 확실한 강점 키워드가 뚜렷할수록 긍정 리뷰의 결집도가 높았다. 이는 쇼핑몰의 브랜딩이 소비자에게 잘 전달되고 있음을 의미한다.

2.  **부정 리뷰의 공통 분모는 '관리'와 '디테일'**:
    * 스타일이 달라도 부정적인 평가는 대부분 **`보풀`, `물빠짐`, `마감(실밥)`, `냄새`** 등 제품의 내구성과 디테일 관리에서 발생했다. 디자인 만족도가 높아도 이러한 품질 이슈는 재구매를 막는 치명적인 요인이 된다.

3.  **사이즈 다양성에 대한 니즈**:
    * **육육걸즈**나 **핫핑**의 경우 `사이즈`, `넉넉함` 키워드가 긍정 토픽의 상위를 차지했다. 이는 온라인 쇼핑에서 '실패하지 않는 사이즈'가 얼마나 중요한 구매 요인인지를 보여준다.

### 6.2. 프로젝트 의의

이 시스템은 방대한 리뷰 데이터를 **자동으로 분류하고 시각화**하여, 쇼핑몰 운영자에게는 **"우리 쇼핑몰의 진짜 문제점과 강점"**을, 소비자에게는 **"내 취향에 맞는 쇼핑몰 탐색 가이드"**를 제공할 수 있다는 점에서 의의가 있다. 향후 리뷰의 작성 시점까지 고려한 시계열 분석을 추가한다면, 시즌별(계절별) 트렌드 변화까지 파악할 수 있는 강력한 도구가 될 것이다.
